## 3. Voice & Tone — confidence: HIGH

**Persona:** Builder-founder. Technical but plain-English. *"Show the receipts. No hype."*

### We Are / We Are Not

| We Are | We Are Not |
|--------|-----------|
| Evidence-first — proof beats vibes | Hype-driven — "revolutionize," "unlock," "synergy" |
| A senior security engineer talking to a peer | A marketer talking at an audience |
| Specific — "hash-chained events," "30 compliance checks" | Vague — "enterprise-grade," "best-in-class," "AI governance" hot-takes |
| Problem-first — lead with the gap, then the fix | Product-first — opening with our name or features |
| Calm authority — confidence doesn't shout | Fear-mongering — "your agents WILL go rogue" |
| Raw > polished when it gets the proof out faster | Over-produced with nothing to show |

### Voice Rules
1. **Lead with the problem,** never the product name.
2. **Show the receipt.** Every claim pairs with proof — a diagram, snippet, hash chain, or framework citation.
3. **Be specific.** "HMAC-SHA256 hash chains" > "advanced cryptography."
4. **Short sentences win.** One idea per sentence.
5. **No fake urgency, no exclamation marks.** Confidence is quiet.
6. **Say "agents," not "AI."** We govern autonomous agents specifically.
7. **Hook first** — earn the first 15 seconds / first line.

## 4. Tone by Context — confidence: HIGH

| Channel | Tone | Length | Structure |
|---------|------|--------|-----------|
| **LinkedIn** | Professional, evidence-led | 3–5 short paras | One insight + one proof artifact + one offer. Claim → receipt → verify. **Never stack links.** |
| **X / Twitter** | Sharp, opinionated | 1-line punches, or 3–5 tweet teardown threads | Hook + insight; definition-style micro-posts |
| **YouTube** | Practitioner-to-practitioner | 6–10 min | Explain one primitive or one incident replay. Hook in 0:00–0:15 |
| **Newsletter (Exhibit A)** | Calm authority, analytical | Long-form | Founder POV, first person, cites specific data/actions, doesn't over-hedge |
| **Blog** | Authoritative, evidence-based | 800–1200 words | Problem + data + solution |
| **Landing page** | Confident, concise | Scannable | Hero claim + proof |
| **Ads (paid)** | Punchy, curiosity-driven | <15 words | Question or bold claim |

**Universal rule:** **one post = one idea = one CTA.**

### Weekly content rhythm (from Content Plan)
- **Mon** — Insight post · **Wed** — Proof asset (diagram/snippet/code) · **Thu** — Teardown story · **Fri** — Offer / CTA

## 5. Terminology — confidence: HIGH

### Preferred
`forensic-grade` · `offline-verifiable` · `tamper-evident` · `evidence bundle` · `hash-chained events` · `signed session attestation` / `DSSE envelope` · `incident replay` · `per-agent identity` / `per-agent keys` · `fail-closed policy` · `audit trail` · `trust root` · `design partner`

### Always Use / Not This
| Use | Not |
|-----|-----|
| AI agents | AI, bots, models |
| Forensics | logging, monitoring, observability |
| Tamper-evident | immutable (we claim *detectability*, not immutability) |
| Fail-closed | secure, protected |
| Per-agent identity | authentication (too generic) |
| Hash-chained events / audit chain | audit log |

### Prohibited (content anti-patterns)
- ❌ Fake urgency
- ❌ Trend-chasing with no message
- ❌ Vague "AI governance" hot-takes
- ❌ Fear-based compliance content with no concrete artifacts
- ❌ "Trust the vendor" framing about *us* (use it only to name the problem)

### Frameworks (reference only what we actually map to)
SOC 2 Type II · NIST AI RMF · EU AI Act
