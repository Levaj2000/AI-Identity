## 1. Positioning

**Primary Tagline:** Every agent gets an identity.

**Elevator Pitch:** Identity and forensics for AI agents.

**Full Description:** AI Identity is the governance infrastructure that gives every AI agent a verified identity, enforceable policies, provable compliance, and forensic accountability. Four pillars — Identity, Policy, Compliance, and Forensics — so enterprises can deploy autonomous agents with confidence.

**Category:** AI Agent Governance / AI Agent Security

## 2. Messaging Hierarchy

### Four Pillars (always in this order)

| Pillar | Question It Answers | One-Liner |
|--------|-------------------|-----------|
| **Identity** | Who is this agent? | Unique API keys, scoped permissions, lifecycle management |
| **Policy** | What can it do? | Fail-closed gateway, per-request policy evaluation |
| **Compliance** | Can we prove it? | Automated evaluators, SOC 2 / NIST / EU AI Act mapping |
| **Forensics** | What happened? | Incident replay, chain verification, forensic export |

### Ad Hooks (rotate all three, let data decide)

- **Forensics lead:** "Your AI agent just made 10,000 API calls. Can you prove which ones were authorized?"
- **Identity lead:** "Your AI agents are sharing one API key. That's a production incident waiting to happen."
- **Compliance lead:** "EU AI Act compliance for your AI agents — in one API call."

### By Channel

| Channel | Tone | Length | Lead With |
|---------|------|--------|-----------|
| **X (Twitter)** | Sharp, technical, opinionated | 1-2 sentences | Hook + insight |
| **LinkedIn** | Professional, thought leadership | 3-5 short paragraphs | Problem + framework |
| **Reddit** | Educational, community-native | Long-form, no pitch | Technical deep dive |
| **Blog** | Authoritative, evidence-based | 800-1200 words | Problem + data + solution |
| **Landing page** | Confident, concise | Scannable sections | Hero claim + proof |
| **Ads (paid)** | Punchy, curiosity-driven | Under 15 words | Question or bold claim |

## 3. Voice & Tone

### We Sound Like

- A senior security engineer explaining something to a peer
- Confident but not arrogant — we know the space, we don't oversell
- Technical but accessible — HMAC chains explained without a PhD
- Problem-first — we lead with the gap, then offer the solution
- Evidence-based — we cite frameworks (SOC 2, NIST, EU AI Act), not adjectives

### We Do NOT Sound Like

- Marketing fluff ("revolutionize", "synergy", "leverage", "unlock")
- Fear-mongering ("your agents WILL go rogue")
- Startup hype ("we're disrupting the AI governance space")
- Vague ("enterprise-grade", "best-in-class", "cutting-edge")

### Voice Rules

1. **Lead with the problem.** Never open with the product name or features.
2. **Be specific.** "HMAC-SHA256 hash chains" > "advanced cryptography." "30 compliance checks" > "comprehensive coverage."
3. **Use questions as hooks.** "Can you prove what your AI agent did at 3:47 AM?" beats "We provide audit trails."
4. **Short sentences win.** Especially in ads and social. One idea per sentence.
5. **No exclamation marks.** Ever. Confidence doesn't shout.
6. **Say 'agents' not 'AI'.** We're specific about what we govern — autonomous AI agents, not AI in general.
7. **Proof over promise.** "Tamper-evident audit chains" > "We keep your agents secure."

## 5. Key Terms & Language

### Always Use

| Term | Not This |
|------|----------|
| AI agents | AI, bots, models |
| Governance | Management, control |
| Forensics | Logging, monitoring, observability |
| Tamper-evident | Immutable (we don't claim immutability — we claim detectability) |
| Fail-closed | Secure, protected |
| Verified identity | Authentication (too generic) |
| Audit chain | Audit log (chain implies cryptographic linking) |
| Four pillars | Features, capabilities |

### Frameworks We Reference

- **SOC 2 Type II** — mention by name, we build toward it
- **NIST AI RMF** — the federal AI risk framework
- **EU AI Act** — the regulatory tailwind
- Never reference frameworks we don't actually map to

### Competitive Positioning

- We are NOT a monitoring tool (Datadog, Grafana, New Relic do that)
- We are NOT an API gateway (Kong, Apigee are generic — we're agent-specific)
- We are NOT an auth provider (Clerk, Auth0 handle human auth — we handle agent auth)
- **Our unique space:** The intersection of identity + policy + compliance + forensics, purpose-built for AI agents

## 7. Content Do's and Don'ts

### Do

- Show code snippets and curl examples in technical content
- Reference real compliance frameworks by name
- Use the four-pillar framework to structure any explanation
- End blog posts with a clear next step (try free, read more, join waitlist)
- Repurpose blog content into social posts (don't create from scratch)

### Don't

- Use stock photography (ever)
- Promise features that aren't shipped
- Compare directly to competitors by name in ads (do it in blog posts only)
- Use jargon without explaining it the first time
- Write content longer than 1200 words unless it's a deep technical guide
- Use emojis in professional content (acceptable in casual X threads only)
